# SmartBook Inventory — Юзкейси

## UC-1: Реєстрація користувача

- **Актор:** Новий користувач (менеджер книгарні)
- **Передумови:** Користувач не зареєстрований у системі
- **Основний сценарій:**
  1. Користувач надсилає POST-запит з username, email, password
  2. Система перевіряє валідність email
  3. Система перевіряє унікальність username та email
  4. Система хешує пароль та створює обліковий запис
  5. Система повертає дані створеного користувача (без пароля)
- **Альтернативні сценарії:**
  - Email невалідний → 400 Bad Request
  - Username або email вже зайнято → 409 Conflict
  - Пароль занадто короткий (< 6 символів) → 400 Bad Request

## UC-2: Вхід у систему (аутентифікація)

- **Актор:** Зареєстрований користувач
- **Передумови:** Користувач має обліковий запис
- **Основний сценарій:**
  1. Користувач надсилає POST з username та password
  2. Система перевіряє облікові дані
  3. Система генерує JWT-токен
  4. Система повертає access_token
- **Альтернативні сценарії:**
  - Невірний username або пароль → 401 Unauthorized

## UC-3: Додавання нової книги на склад

- **Актор:** Авторизований менеджер
- **Передумови:** Користувач аутентифікований (має валідний JWT)
- **Основний сценарій:**
  1. Менеджер надсилає POST з даними книги (title, author, isbn, price, quantity, genre)
  2. Система валідує дані:
     - title не порожній
     - ISBN відповідає формату (13 цифр)
     - price > 0
     - quantity >= 0
  3. Система перевіряє, що ISBN унікальний
  4. Система створює запис книги
  5. Система повертає 201 Created з даними книги
- **Альтернативні сценарії:**
  - Невалідні дані → 400 Bad Request з деталями помилки
  - ISBN вже існує → 409 Conflict
  - Немає JWT → 401 Unauthorized

## UC-4: Перегляд списку книг

- **Актор:** Авторизований менеджер
- **Передумови:** Користувач аутентифікований
- **Основний сценарій:**
  1. Менеджер надсилає GET-запит (опціонально з фільтрами: genre, author, пошук за title)
  2. Система повертає список книг з пагінацією
- **Альтернативні сценарії:**
  - Немає JWT → 401 Unauthorized
  - Немає результатів → 200 OK з порожнім списком

## UC-5: Перегляд деталей книги

- **Актор:** Авторизований менеджер
- **Передумови:** Книга існує в системі
- **Основний сценарій:**
  1. Менеджер надсилає GET-запит з ID книги
  2. Система повертає повні дані книги
- **Альтернативні сценарії:**
  - Книга не знайдена → 404 Not Found

## UC-6: Оновлення даних книги

- **Актор:** Авторизований менеджер
- **Передумови:** Книга існує в системі
- **Основний сценарій:**
  1. Менеджер надсилає PUT з оновленими даними
  2. Система валідує нові значення (ті ж інваріанти, що й при створенні)
  3. Система оновлює запис
  4. Система повертає 200 OK з оновленими даними
- **Альтернативні сценарії:**
  - Книга не знайдена → 404 Not Found
  - Невалідні дані → 400 Bad Request
  - Спроба змінити ISBN на вже зайнятий → 409 Conflict

## UC-7: Видалення книги

- **Актор:** Авторизований менеджер
- **Передумови:** Книга існує, не має активних (pending) замовлень
- **Основний сценарій:**
  1. Менеджер надсилає DELETE-запит з ID книги
  2. Система перевіряє відсутність активних замовлень на цю книгу
  3. Система видаляє книгу
  4. Система повертає 204 No Content
- **Альтернативні сценарії:**
  - Книга не знайдена → 404 Not Found
  - Є активні замовлення → 409 Conflict

## UC-8: Створення замовлення (продаж книг)

- **Актор:** Авторизований менеджер
- **Передумови:** Книги існують і є в достатній кількості на складі
- **Основний сценарій:**
  1. Менеджер надсилає POST з переліком книг та кількостей
  2. Система перевіряє кожну позицію:
     - Книга існує
     - **Інваріант: кількість на складі >= кількість у замовленні** (не можна продати більше, ніж є)
     - Кількість у замовленні > 0
  3. Система зменшує quantity кожної книги на складі
  4. Система створює замовлення зі статусом "completed"
  5. Система повертає 201 Created з деталями замовлення (включно з total_price)
- **Альтернативні сценарії:**
  - Книга не знайдена → 404 Not Found
  - Недостатньо книг на складі → 409 Conflict з зазначенням, яка книга і скільки доступно
  - Порожній список позицій → 400 Bad Request

## UC-9: Перегляд списку замовлень

- **Актор:** Авторизований менеджер
- **Передумови:** Користувач аутентифікований
- **Основний сценарій:**
  1. Менеджер надсилає GET-запит (опціонально з фільтром по статусу)
  2. Система повертає список замовлень з пагінацією
- **Альтернативні сценарії:**
  - Немає JWT → 401 Unauthorized

## UC-10: Перегляд деталей замовлення

- **Актор:** Авторизований менеджер
- **Передумови:** Замовлення існує
- **Основний сценарій:**
  1. Менеджер надсилає GET з ID замовлення
  2. Система повертає деталі замовлення з переліком книг, кількостей та цін
- **Альтернативні сценарії:**
  - Замовлення не знайдено → 404 Not Found

## UC-11: Поповнення запасів (Restocking)

- **Актор:** Авторизований менеджер
- **Передумови:** Книга існує в системі
- **Основний сценарій:**
  1. Менеджер надсилає PATCH із зазначенням book_id та кількості для додавання
  2. Система додає кількість до поточного запасу
  3. Система повертає оновлені дані книги
- **Альтернативні сценарії:**
  - Книга не знайдена → 404 Not Found
  - Кількість <= 0 → 400 Bad Request
